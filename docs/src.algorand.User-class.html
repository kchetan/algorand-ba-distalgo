ByzantineGossip(self, msg, to=None)

source code 

The Byzantine gossip has two parts. If the message is a proposal then the user proposes two different blocks to its neighbours. If the message is a committee gossip then the user either sends a malicious block or an empty block on a random basis.

Parameters:
  • msg (tuple) - The message tuple that has to be send to the neighbours in a byzantine way.
  • to (set or None.) - The set of users the message has to be sent to.

VerifySort(self, proc_id, role, round_num, step)

source code 

Sends a request to controller to verify that a given proc_id is actually a committee member. A committee member check will return the number of votes that member can cast (or 0 if not a member). A proposer check will return 1 if user is a proposer in that round, and 0 if not. Abstracting Cryptographic part in the Algorithm 2 (page 6) - https://people.csail.mit.edu/nickolai/papers/gilad-algorand-eprint.pdf

Parameters:
  • proc_id (User) - Process ID of original message sender
  • role (str) - Role for which to be checked
  • round_num (int) - Round number for which to be checked
  • step (int or str) - Step number for which to be checked
Returns: int
1/0 in case of Proposer Check, and num_votes/0 in case of Committee Member Check

ProcessMsg(self, ctx, msg)

source code 

ProcessMsg() returns not just the value contained in the message, but also the number of votes associated with that value. If the message was not from a chosen committee member, ProcessMsg() returns zero votes. If the committee member was chosen several times the number of votes returned by ProcessMsg() reflects that. As in Algorithm 6 ( page 9) - https://people.csail.mit.edu/nickolai/papers/gilad-algorand-eprint.pdf

Parameters:
  • ctx (Context) - It captures the current state of the ledger
  • msg (Tuple) - A tuple of (round_num, step, gossip_initiator, hash_prev_block, value)
Returns: int, User, H(Block)
Returns the number of votes that can be cast by the User named initiator_id, and the value that has been voted for by that User.

CommitteeVote(self, context, round, step, value)

source code 

The procedure checks if the user is selected for the committee in a given round and step of BA. The CommitteeVote procedure to check if the user is chosen to participate in the committee. If the user is chosen for this step, the user gossips the message containing the value passed to CommitteeVote, which is typically the hash of some block. As in Algorithm 4 ( page 8) - https://people.csail.mit.edu/nickolai/papers/gilad-algorand-eprint.pdf

Parameters:
  • context (Context) - It captures the current state of the ledger
  • round (int) - The current round of the BA.
  • step (int) - The current step of BA that is in progress.
  • value (H(Block)) - The highest priority block's hash that has been received by the user.
Returns: NoneType
None

CountVotes(self, context, round_num, step, T_value, tau_value, lambda_value)

source code 

This procedure reads messages that belong to the current round and step. It processes the votes by calling the ProcessMsg() procedure for every message , which ensures that the vote is valid. Count the number of votes for each proposed blocks hash value and returns the block if the value is more than a threshold. As in Algorithm 5 ( page 9) - https://people.csail.mit.edu/nickolai/papers/gilad-algorand-eprint.pdf Note: used yield points instead of the message.iterator as in the paper.

Parameters:
  • context (Context) - It captures the current state of the ledger
  • round_num (int) - The current round of the BA.
  • step (int) - The current step of BA that is in progress.
  • T_value (float) - Threshold of tau value for BA*
  • tau_value (int) - Expected number of block or values.
  • lambda_value (int) - Timeout for receiving a block
Returns: H(Block)
Returns the hash of the block that has maximum votes.

BinaryBA(self, context, round, block_hash)

source code 

In the procedure, it reaches consensus on one of two values: either the hash passed to BinaryBA() or the hash of the empty block. BinaryBA() relies on Reduction() to ensure that at most one non-empty block hash is passed to BinaryBA() by all honest users. As in Algorithm 8 ( page 10) - https://people.csail.mit.edu/nickolai/papers/gilad-algorand-eprint.pdf

Parameters:
  • context (Context) - It captures the current state of the ledger
  • round (int) - The current round of the BA.
  • block_hash (H(Block)) - The highest priority block's hash that has been received by the user.
Returns: H(Block)
Returns the hash of the block that has been chosen.
